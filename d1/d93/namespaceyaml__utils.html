Mat &  M

Load an optional matrix from YAML by specifying whole columns.

The YAML node is expected to be a mapping from integer column indices (0-based) to sequences holding the elements of that column:

bdhyde:
0: [c0_r0, c0_r1, ..., c0_rN]
1: [c1_r0, c1_r1, ..., c1_rN]

For each column key:

  • If the column index is out of range, it is ignored.
  • If the value is a sequence, up to M.rows() entries are read.
  • If the sequence is shorter than M.rows(), the remaining rows keep their current values (defaults set in C++).
模板参数
MatMatrix type with:
  • int rows() const
  • int cols() const
  • Scalar& operator()(int,int)
  • using Scalar = ...; Typically an Eigen::Matrix<Scalar, Rows, Cols>.
参数
nYAML mapping node that may contain the key.
keyMapping key describing the matrix columns.
MDestination matrix to modify column-wise.

Load an optional scalar value from a YAML mapping node.

If the given key exists in the mapping n, the destination dst is overwritten with the converted value. If the key does not exist, dst is left unchanged (i.e., its C++ default or previously set value is preserved).

模板参数
TScalar type to load (e.g. int, double, bool, std::string).
参数
nYAML mapping node that may contain the key.
keyMapping key to look up.
dstDestination variable to update if the key is present.

Load an optional sparse std::array from a YAML mapping.

This overload of load_vector_optional handles std::array<T, N>, which provides:

  • std::size_t size() const
  • T& operator[](std::size_t)

The YAML structure is the same as for Eigen vectors:

flags:
0: true
3: [false, true] # sets index 3 and 4

Indices outside [0, N) are silently ignored.

模板参数
TElement type of the array.
NNumber of elements in the array.
参数
nYAML mapping node that may contain key.
keyMapping key describing sparse updates.
vDestination std::array to modify.

template<typename Vec >
void yaml_utils::load_vector_optional ( const YAML::Node &  n,
const char *  key,
Vec &  v 
)

Load an optional sparse Eigen-style vector from a YAML mapping.

This overload is intended for Eigen vector-like types that provide

  • int size() const
  • Scalar& operator()(int)
  • using Scalar = ...

The YAML node key is expected to be a mapping from integer indices to either a scalar or a sequence:

cfutz:
0: [0.0, 0.0, 0.0] # write 3 elements starting at index 0
8: [1.0, 0.0] # write 2 elements starting at index 8
18: 1.0 # write a single element at index 18

For each entry:

  • if the value is a sequence, elements are written starting at start,
  • if the value is a scalar, only the single index start is updated.

Indices outside [0, v.size()) are silently ignored.

Typical usage:

  • Initialize v with size and defaults in the constructor;
  • Call this function from a load() method to optionally override selected entries from the YAML file.
模板参数
VecEigen-style vector type.
参数
nYAML mapping node that may contain key.
keyMapping key describing sparse updates.
vDestination vector-like container to modify.
